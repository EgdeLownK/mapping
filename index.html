❓ Documentation — Architecture Mapping

🎯 À quoi ça sert ?

Cet outil sert à cartographier l'architecture complète d'une application sous forme de schéma structuré. L'objectif principal est de produire un fichier 🤖 Export IA qu'une intelligence artificielle peut lire pour comprendre intégralement comment l'app est construite — sans avoir accès au code.

Le schéma décrit ce qui existe dans l'interface (les composants et leurs éléments) et comment ils sont reliés (les flux de navigation). Une IA peut ainsi répondre à des questions sur l'architecture, suggérer des évolutions, détecter des incohérences ou générer du code en contexte.

🗂️ Les 4 concepts fondamentaux

📄 Pages — Une page correspond à une section logique du site ou de l'app : par exemple "Vitrine publique", "Authentification", "Espace connecté". Ce n'est pas forcément une URL unique — c'est un contexte d'affichage de haut niveau qui regroupe plusieurs composants.

⬜ Tables (Composants) — Chaque table représente un composant d'interface : un bloc autonome et identifiable comme un menu de navigation, une page de profil, un formulaire de connexion, un dashboard, une sidebar. Un composant appartient à une page et peut contenir de nombreux éléments.

· Éléments — Ce sont les briques internes d'un composant : un bouton CTA, un champ de saisie, un onglet, un lien, une zone média... Chaque élément a un type précis (défini dans le widget Types) et peut être la source d'une relation.

🔗 Relations — Une relation (flèche) part d'un élément précis d'un composant et pointe vers un composant cible. Elle modélise un flux de navigation : quel bouton ouvre quelle page, quel formulaire redirige où, quel onglet affiche quel sous-composant.

🤖 Export IA — comment ça marche

Le bouton 🤖 Export IA génère un .json épuré, sans aucune donnée visuelle (pas de couleurs, pas de positions). Le fichier expose :

  • Le dictionnaire des types d'éléments : ce que chaque tag signifie fonctionnellement
  • Les composants avec leurs éléments, leur page parente et leur composant parent si applicable
  • Les flux : quel élément déclenche quelle transition, vers quel composant, et de quel type d'interaction il s'agit

💡 Plus les champs Description des composants sont remplis, plus l'IA comprendra le rôle de chaque bloc sans ambiguïté.
